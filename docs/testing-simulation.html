O Pipeline de Validação

Para garantir que a lógica de Kafka Streams e o roteamento de mensagens estejam funcionando, utilizamos um script de automação que simula comportamentos reais de usuários.

Script: simulate-traffic.sh
Objetivo: Enviar rajadas de transações via curl para o endpoint de ingestão.

Cenário de Teste: Rajada de Transações (Burst)

A regra de fraude atual define que mais de 3 transações em menos de 2 minutos para a mesma conta é considerado comportamento suspeito.

  1. O script envia 1 transação normal para USER-001 (Aprovada).
  2. O script envia 4 transações consecutivas para ACC-FRAUD-01.
  3. Resultado esperado: As 3 primeiras da rajada são validadas, a 4ª dispara um alerta.

Execução do Script

chmod +x simulate-traffic.sh
./simulate-traffic.sh

Verificação no AKHQ

Após a simulação, os dados podem ser analisados visualmente:

  • transactions-raw: Contém os 5 eventos originais (JSON bruto).
  • transactions-validated: Contém os 4 eventos aprovados pelo processador.
  • transactions-alerts: Contém o 5º evento (a 4ª transação da conta suspeita) marcado como REJECTED.
