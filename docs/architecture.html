Fluxo de Dados

O fluxo de dados segue uma jornada do cliente até os tópicos de saída finais.

  1. O Client faz um POST para /api/transactions.
  2. O Controller valida o JSON da requisição.
  3. O Service gera metadados (ID, Timestamp, Status PENDING).
  4. A transação é publicada no tópico transactions-raw.
  5. O Kafka Streams consome de transactions-raw, analisa a janela e roteia para transactions-validated ou transactions-alerts.

Componentes do Sistema

  • Model (Transaction): Classe central contendo o estado da transação.
  • DTO (TransactionRequestDTO): Objeto simplificado para recepção de requisições.
  • Service: Core de negócio para criação de transações.
  • KafkaConfig: Responsável pelo KafkaTemplate e criação automática de tópicos.
  • FraudStreamProcessor: A topologia Kafka Streams (Core Inteligente).
