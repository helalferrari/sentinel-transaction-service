Pirâmide de Testes

O projeto utiliza múltiplas camadas de testes para garantir a integridade funcional e técnica.

1. Testes Unitários (Mockito)

Utilizados para validar a lógica de negócio pura no TransactionService e no AlertConsumerService. O Kafka e o WebSocket são mockados para garantir velocidade.

2. Web Slice Tests (MockMvc)

Validam o TransactionController, garantindo que os endpoints respondam nos status corretos e que a validação de campos (Bean Validation) funcione.

Nota: O Kafka Streams é desabilitado nestes testes via perfis do Spring (@ActiveProfiles("test")).

3. Kafka Streams Topology Test

Utiliza o TopologyTestDriver. É a camada mais técnica, validando a janela de tempo e a lógica de fraude sem a necessidade de um broker Kafka real.

Como Executar

Para rodar todos os testes do projeto:

./mvnw test
